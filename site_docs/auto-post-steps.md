# 自動發文完整步驟

這份文件專門記錄如何使用自動化腳本來發布新文章。

## 1. 撰寫草稿
在專案根目錄中找到 `draft.md` 檔案，並使用 Markdown 語法撰寫您的文章內容。

請務必在文章的最上方加上文章的屬性設定 (Frontmatter)：
```yaml
---
title: 您的文章標題
date: 2026-06-04
tags:
  - "標籤1"
categories:
  - "分類1"
---
```

*(💡 提示：如果文章中有程式碼，就算忘記標註語言（如 python / js），系統也會自動幫您猜測並上色！)*

## 2. 執行自動發文腳本
寫完內容後，請打開終端機 (Terminal)，確認路徑在專案根目錄，然後輸入以下指令：

```bash
pnpm run auto-post draft.md
```
> **注意**：這支程式不僅會自動為您的文章配上封面圖片並壓縮，**還會額外請 AI 閱讀您的文章，自動在重點字句加上螢光筆標籤 `<mark>`**！最後會將整篇文章打包轉移到正式的 `src/content/posts` 資料夾中。

## 3. 上傳圖片至 Cloudflare R2
因為我們的網站架構已經全面升級為雲端圖片庫，所以請執行以下指令，將草稿中產生的封面圖片與內文圖片自動上傳到 R2 雲端：

```bash
pnpm run upload-images
```

## 4. 推送與部署 (帶有 Vercel 進度條)
腳本執行成功且圖片上傳完畢後，文章已經在本地端準備完畢。您不再需要手動輸入多行 Git 指令，只要執行專屬的發布指令：


```bash
pnpm run publish "新增了一篇超棒的文章"
```

這個指令會自動：
1. 將所有變更加入並推送到 GitHub (`git add`, `git commit`, `git push`)。
2. 終端機會顯示一個 **Vercel 自動部署進度條**，您可以即時看著狀態從 `[Building]` 變成 `[Ready]`。
3. 部署完成後，會直接印出這次更新的正式網址！

---
**其他備用方式**
如果您沒有使用 GitHub 自動部署，也可以直接透過 Vercel CLI 手動發布：
```bash
npx vercel --prod
```
